In simple terms, Git is a tool that helps developers keep track of changes in their code and work together. It's like a system for managing different versions of code, so you can easily see what changed, go back to earlier versions, and collaborate with others.

Basically, Git is a distributed version control system (DVCS) that helps developers manage and track changes to their code, letting multiple people work on a project at the same time. It keeps a history of all the changes and makes it easier to coordinate updates, making teamwork a lot smoother.

From the Git website: It is a free and open source distributed version control system designed to handle everything from small to very large projects with speed and efficiency. Git is easy to learn and has a tiny footprint with lightning fast performance. It outclasses SCM tools like Subversion, CVS, Perforce, and ClearCase with features like cheap local branching, convenient staging areas, and multiple workflows.


Key aspects


  • Version Control

    Git keeps track of changes made to files over time. This allows developers to revert files or projects back to previous versions, compare changes over time, and see who last modified something that might be causing a problem.

  • Distributed System

    Unlike traditional version control systems like Subversion or CVS, Git is distributed. Every developer has a full copy of the repository, including its entire history. This allows for greater flexibility and offline work.

  • Branching and Merging

    Git makes it easy to create, manage, and merge branches. Branching allows developers to diverge from the main project to work on separate features or fixes independently, then merge their changes back into the main branch when ready.

  • Commit

    Changes in Git are recorded in snapshots called commits. Each commit is a full snapshot of the files at a certain point in time, not just the differences from the previous commit.

  • Repository

    A Git repository contains all the project files and the entire revision history. Repositories can be local (on your computer) or remote (on a server like GitHub, GitLab, or Bitbucket).

  • Collaboration

    Git enables collaboration among multiple developers. Changes can be pulled from remote repositories, reviewed, and merged into local repositories. It also facilitates code reviews, continuous integration, and deployment workflows.

Getting Started with Git

  • Install Git Download and install Git from git-scm.com.

  • Create a Repository Use git init to start a new project or git clone [url] to copy an existing one.

  • Make Changes Edit your files as needed.

  • Track Changes Use git add to stage changes and git commit to save them.

  • Share Your Work Use git push to upload your changes to a remote repository and git pull to get updates from others.

Common Git Commands

  • git init Initializes a new Git repository.

  • git clone [url] Clones an existing repository from a remote server.

  • git add [file] Stages files to be committed.

  • git commit -m "message" Commits staged changes with a message.

  • git push Pushes local commits to a remote repository.

  • git pull Fetches and integrates changes from a remote repository.

  • git status Shows the status of changes in the working directory.

  • git log Displays the commit history.

  • git branch Lists branches, creates a new branch, or deletes a branch.

  • git checkout [branch] Switches to a different branch.

  • git merge [branch] Merges changes from one branch into another.

Let's Code

Just a heads up: This article only goes over the basics of Git and the most commonly used commands. So, I'm going to assume you already know how to create a repository on one of the remote servers we talked about earlier.


Please take into consideration that the following commands are performed using GitBash (Windows users) or Terminal (MacOs users) 


                                
									// let's clone a repository previously created in GitHub
									// using git clone; this command will clone desired repo into
									// a folder with the same repo name in relative to your current path
									git clone git@github.com:[your-user]/lets-code.git

									// changing directory to the one created by git clone command
									// using 'cd' command
									cd [newly created directory name]

									// once 'inside' the directory we can list the branches with 'git branch' command
									// at this point we should only have the 'main' branch
									git branch

									// let's create a new branch named 'feature/add-page-and-style'
									// with 'git checkout-b'
									// this command in combination of the '-b' flag will perform 2 actions
									// 1. create the branch (git branch feature/add-page-and-style)
									// 2. checkout to the branch (git checkout feature/add-page-and-style)
									git checkout -b feature/add-page-and-style

									// let's create the index.html file with the 'touch' command
									touch index.html

									// we can check the current status of our work by using the 'git status' command
									git status

									// now we can track the changes with the 'git add' command
									// we are going to use the '-A' to include all current changes at once
									git add -A

									// after adding the files to be tracked we can commit the changes
									// using the 'git commit' command
									// we will use the '-m' flag to write a meaningful comment in the same line
									git commit -m "add: index html file"

									// now we can add the style css file
									// using same workflow as adding the index.html file
									touch style.css

									git status
									git add -A
									git commit -m "add: style css file"

									// once we are happy and compliant with the feature
									// we can push the changes to any of the remote servers using the 'git push' command
									git push

									// once we have successfully uploaded the desired changes to GitHub
									// we need to checkout to the 'main' branch to reflect those changes
									// in the main branch using git checkout and git pull
									git checkout main

									// git pull will 'fetch' new changes in remote and 'merge' them in
									// our current branch
									git pull

Conclusion

Git has become a fundamental tool in modern software development, fostering collaborative and efficient development practices. It is a powerful tool that becomes easier to understand the more you use it. Start with basic commands and gradually explore more advanced features as you become comfortable.
